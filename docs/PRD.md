## 1. 專案概述

### 1.1 背景與動機

算命、占卜、抽籤是華人文化中深受歡迎的傳統活動，許多人習慣在求職、考試、感情、健康等人生重要時刻尋求命理指引。然而，前往實體廟宇或算命攤位並不總是方便。本系統旨在打造一個線上算命平台，讓使用者隨時隨地都能進行算命、占卜與抽籤，獲取命理建議與心靈指引。

### 1.2 目標用戶

- **一般大眾**：對命理、運勢有興趣，想在日常生活中隨手占卜的使用者
- **年輕族群**：喜歡趣味互動體驗，願意在線上嘗試抽籤、塔羅占卜等新鮮玩法的學生與上班族
- **尋求指引者**：面臨人生抉擇（求職、感情、考試等），希望獲得心靈慰藉或參考方向的人
- **信仰文化愛好者**：對傳統廟宇文化有興趣，希望在線上體驗擲筊、抽籤等儀式的使用者

### 1.3 核心價值主張

> **隨時隨地，線上求籤問卜，獲取命理指引與心靈慰藉。**

- 🎋 提供多種算命方式（抽籤、擲筊、塔羅牌占卜），滿足不同需求
- 💾 算命結果可儲存，方便使用者日後回顧
- 🙏 捐香油錢功能，保留傳統文化中感恩還願的精神
- 🌙 沉浸式的神秘風格 UI，增添儀式感與趣味性

## 2. 功能需求

### 功能一：抽籤（求籤）

> **作為**一位使用者，**我希望**能在線上進行抽籤，隨機抽取一支籤詩，**以便**獲取運勢指引與建議。

**詳細說明：**
- 使用者可選擇求籤類別（綜合運勢、感情、事業、學業、財運、健康）
- 系統隨機產生一支籤（上上籤、上籤、中籤、下籤、下下籤）
- 顯示籤詩內容、白話解釋與建議
- 搭配動畫效果（搖籤筒、擲筊確認等視覺呈現）

### 功能二：塔羅牌占卜

> **作為**一位使用者，**我希望**能進行線上塔羅牌占卜，**以便**從塔羅牌的象徵意義中獲取建議與啟示。

**詳細說明：**
- 使用者先選擇占卜主題（愛情、事業、人際、當日運勢）
- 提供牌陣選擇（單張牌、三張牌-過去/現在/未來）
- 使用者點擊翻牌，顯示牌面與解讀
- 每張牌包含圖片、牌名、正位/逆位含義

### 功能三：擲筊

> **作為**一位使用者，**我希望**能在線上擲筊，詢問是非題，**以便**獲得神明的指示（聖筊、笑筊、怒筊）。

**詳細說明：**
- 使用者輸入一個是非問題
- 點擊「擲筊」按鈕後，系統隨機產生結果
- 結果包含：聖筊（一正一反，表示同意）、笑筊（兩正面，表示在笑）、怒筊（兩反面，表示不同意）
- 搭配擲筊動畫效果

### 功能四：算命結果儲存與歷史紀錄

> **作為**一位使用者，**我希望**能將每次的算命結果儲存起來，日後可以查看歷史紀錄，**以便**回顧過往的運勢變化與建議。

**詳細說明：**
- 每次算命結果自動儲存到資料庫
- 使用者可在「歷史紀錄」頁面查看所有過去的算命結果
- 支援依日期、類型（抽籤/塔羅/擲筊）篩選
- 可刪除不需要的紀錄

### 功能五：捐香油錢

> **作為**一位使用者，**我希望**在算命之後能捐香油錢表達感謝，**以便**延續傳統文化中還願感恩的精神。

**詳細說明：**
- 提供捐款功能頁面，顯示香油錢箱意象
- 使用者可選擇固定金額（10 元、50 元、100 元、500 元）或自訂金額
- 記錄捐款歷史（日期、金額）
- 顯示感謝動畫與祝福語
- **注意：** 本次練習僅模擬捐款流程，不串接真實金流

### 功能六：每日運勢

> **作為**一位使用者，**我希望**每天打開系統就能看到今日運勢，**以便**快速掌握當日的吉凶與注意事項。

**詳細說明：**
- 首頁顯示「今日運勢」區塊
- 根據系統日期自動產生當日運勢（幸運色、幸運數字、整體運勢星級、一句話建議）
- 每日內容不同，增加使用者回訪動力

### 功能七：算命方式總覽與導覽

> **作為**一位使用者，**我希望**在首頁就能清楚看到所有可用的算命方式，**以便**快速選擇我想要的占卜類型。

**詳細說明：**
- 首頁以卡片式設計展示所有算命方式（抽籤、塔羅、擲筊）
- 每個卡片包含圖示、名稱、簡短說明
- 點擊卡片直接進入對應功能頁面

## 3. 非功能需求

### 3.1 技術限制

| 項目   | 技術選擇                  |
| ------ | ------------------------- |
| 後端   | Python + Flask            |
| 模板   | Jinja2（Flask 內建）      |
| 資料庫 | SQLite                    |
| 前端   | HTML + CSS + JavaScript   |

### 3.2 效能考量

- 頁面載入時間應在 2 秒以內
- 資料庫查詢應有適當索引，避免慢查詢
- 靜態資源（圖片、CSS、JS）應組織良好，避免過大的檔案影響載入速度

### 3.3 安全考量

- 使用者輸入（問題、自訂金額等）需進行驗證與過濾，防止 XSS 與 SQL Injection
- 捐款功能僅為模擬，不處理真實金融資訊
- 使用 Flask 內建的 CSRF 保護機制

### 3.4 使用者體驗

- 介面風格以「神秘、典雅、東方」為主題，採用深色系配色搭配金色點綴
- 互動元素需搭配動畫效果（籤筒搖晃、筊杯翻轉、塔羅翻牌），增加儀式感
- 支援 RWD 響應式設計，在手機與桌面皆可正常使用
